Introduction

As I'm already familiar with Javascript and eager to dive into React, I've taken this project as an opportunity to outline some beginner React concepts.

But first, what is React? On React's official site, it is described simply as a "JavaScript library for building user interfaces." In other words, React is concerned with rendering data to the DOM and updating that data as necessary, based on user interaction.

React highlights three qualities of their library on their homepage, quoted here:

  • Declarative: "React makes it painless to create interactive UIs. Design simple views for each state in your application, and React will efficiently update and render just the right components when your data changes. Declarative views make your code more predictable and easier to debug."
  • Component-Based: "Build encapsulated components that manage their own state, then compose them to make complex UIs. Since component logic is written in JavaScript instead of templates, you can easily pass rich data through your app and keep state out of the DOM."
  • Learn Once, Write Anywhere: "We donâ€™t make assumptions about the rest of your technology stack, so you can develop new features in React without rewriting existing code. React can also render on the server using Node and power mobile apps using React Native."

Hello World

What better place to start than "Hello World"? Here's the code:

                            
                                ReactDOM.render(
                                    <h1>Hello, world!</h1>,
                                    document.getElementById('root')
                                );

As you might guess, this simply displays a heading that reads "Hello, world!"

JSX

In the second line of our example above, you can see the code <h1>Hello, world!</h1>.

It looks just like html, but it's actually JSX, which React describes as a "syntax extension" to Javascript.

Why is JSX used? The creators of React decided it doesn't make sense to separate markup and logic, since these are naturally presented together.

For example, what if we wanted to say hello to a specific person based on some logic? We could instead create a line of JSX like this, essentially including a variable: <h1>Hello, {name}</h1>.

In fact, you can put any valid Javascript expression in JSX, so long as it is inside curly braces.

Rendering Elements

Now let's look at the first line of our "Hello World" example (ReactDOM.render) to begin understanding how rendering works. Rendering is, unsurprisingly, done with the render() function. The first argument or element is what we want to render, and the second is where we want to render it. Specifically, this is rendering to the element with the #root ID.

Here are two important things to note about rendering:

  • React elements are immutable. If you want to update the UI, you must crate a new element and render it.
  • React only updates what is needed by comparing the new element to the previous one.

Components and Props

In React, a component is essentially a Javascript function, which takes props as arguments. Here is an example that renders "Hello, Sara" to the DOM:

                            
                                function Welcome(props) {
                                    return <h1>Hello, {props.name}</h1>
                                }
                                    
                                const element = <Welcome name="Sara" />
                                ReactDOM.render(
                                    element,
                                    document.getElementById('root')
                                );

The first chunk of code is our Welcome component, which returns some JSX that includes an expression from the props received. But what exactly is going on with props.name? Where is the name coming from? As you can see in the next line defining the constant element, the name is defined with the string "Sara". You can have multiple props with any variable names you choose.

The rest of this example is familiar territory, rendering the element to the #root.
